# Gazetor — сводка новостей "в стиле Джарвиса"

Бесплатный бот, который дважды в день (11:00 и 21:00) присылает краткую
нейтральную сводку новостей из выбранных вами Telegram-каналов через RSS
(RSSHub). Реклама и малозначимые посты отфильтровываются, точка зрения — нейтральная.

Всё можно настроить с телефона: через приложение Telegram и браузер/приложение GitHub.

## Шаг 1. Создать Telegram-бота

1. В Telegram откройте **@BotFather**.
2. Отправьте `/newbot`, задайте имя (например `Gazetor`) и username, оканчивающийся на `bot`.
3. Получите **токен** вида `123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxx`. Сохраните его.
4. Напишите своему новому боту любое сообщение (например `/start`) — иначе он не сможет писать вам первым.

## Шаг 2. Узнать свой chat_id

1. Откройте в браузере (замените `<TOKEN>` на свой токен):
   `https://api.telegram.org/bot<TOKEN>/getUpdates`
2. В ответе найдите `"chat":{"id": 123456789, ...}` — это ваш **CHAT_ID**.
   (Если ответ пустой — убедитесь, что уже написали боту в шаге 1.4.)

## Шаг 3. Получить бесплатный ключ Google Gemini (для суммаризации)

1. Зайдите на **aistudio.google.com** с телефона.
2. Войдите через Google-аккаунт → "Get API key" → "Create API key".
3. Сохраните ключ. Бесплатного лимита Gemini 1.5 Flash хватит с большим запасом на 2 запуска в день.

## Шаг 4. Создать репозиторий на GitHub

1. Установите приложение **GitHub** (или используйте github.com в браузере).
2. Создайте новый **приватный** репозиторий, например `gazetor`.
3. Загрузите в него 3 файла из этого проекта:
   - `fetch_and_send.py`
   - `requirements.txt`
   - `.github/workflows/digest.yml` (при загрузке через веб-интерфейс "Add file → Create new file" укажите полный путь `.github/workflows/digest.yml` — папки создадутся автоматически)

## Шаг 5. Добавить секреты и переменные

В репозитории: **Settings → Secrets and variables → Actions**

Вкладка **Secrets** → "New repository secret", добавить три секрета:
- `BOT_TOKEN` — токен из шага 1
- `CHAT_ID` — id из шага 2
- `LLM_API_KEY` — ключ из шага 3

Вкладка **Variables** → "New repository variable":
- `CHANNELS` со значением:
  `odessa_infonews,truexanewsua,slav_kram,hiaimedia`

## Шаг 6. Проверить время

Расписание в `digest.yml` рассчитано на летнее время Украины/Москвы (UTC+3):
`08:00 UTC` = `11:00`, `18:00 UTC` = `21:00`.

Если это не ваш часовой пояс — измените значения `cron` в файле
`.github/workflows/digest.yml` (первое число — минуты, второе — час в UTC).

## Шаг 7. Проверить вручную

В репозитории откройте вкладку **Actions → Gazetor Digest → Run workflow**
(кнопка справа) — запустит скрипт немедленно, не дожидаясь расписания.
Если всё настроено верно, через 10–20 секунд сообщение придёт в Telegram.

## Как это устроено внутри

1. `fetch_and_send.py` для каждого канала из `CHANNELS` обращается к
   `https://rsshub.app/telegram/channel/<канал>` — публичному RSS-фиду
   Telegram-канала — и забирает последние посты.
2. Все посты собираются в один текст и отправляются в Gemini с инструкцией
   отфильтровать рекламу, убрать дубли, сохранить нейтральную точку зрения
   и написать сводку в стиле Джарвиса.
3. Готовая сводка отправляется вам в Telegram через Bot API.
4. GitHub Actions запускает этот скрипт по расписанию — в облаке, бесплатно,
   без необходимости держать телефон или компьютер включённым.

## Важные ограничения

- Используется публичный сервер `rsshub.app` — он бесплатный, но иногда
  бывает медленным или временно недоступным (rate limit). Если сводки
  начнут регулярно пропадать — можно указать секрет/переменную `RSSHUB_BASE`
  с адресом альтернативного зеркала RSSHub или собственного инстанса.
- Работает только с каналами, для которых RSSHub может построить RSS
  (обычно это публичные каналы).
- GitHub Actions на приватном репозитории даёт ~2000 бесплатных минут
  в месяц — двух коротких запусков в день более чем достаточно.
- Бесплатный лимит Gemini может со временем измениться — при необходимости
  можно заменить на другой бесплатный LLM API, поменяв функцию
  `summarize_with_gemini` в `fetch_and_send.py`.
